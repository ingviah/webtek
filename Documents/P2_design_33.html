Purpose, goals, and audience

The purpose of the site is to unite fans and players while promoting Nonstop FK. The site should contain all the information about the football team. It will contain a list of all upcoming and played matches, and information about the players. The intended users are players of Nonstop FK as well as the supporters of the team. Both the players and the supporters have a need to follow the team.

Navigation Structure

As you can see on the storyboard above all html files are linked to each other. That is because there will be a navigation bar present on every page. From the navigation bar you can access the team page and fixtures page, and can return to the homepage by clicking on the logo present in the navbar.

Content

homepage.html

The homepage is the first page the user will see when entering the website. The goal for this page is to provide the user with useful information. We decided to make the next scheduled match visible in the homepage for easy access for the user. It is a simple text layout with who the team is playing against, where and when. To the left we have provided a table of the position of the teams in the league, because this is something we think the typical user will be interested in, and therefore it should be easily accessed. To keep the users updated on the latest news regarding the football team we provide a grid with pictures, a bold heading, and the first characters of the text. In this way the user can get an overview of the news articles, and click on what they want to read more about. Also, there is a question/feedback input box in the bottom left corner. We want it to be easy for the user to contact the team. The navigation bar is visible at every page on the website. The navigation bar has three sections: the logo, “Kampoppsett”, and “Laget”. Through these sections the user gets access to every page of the website. When clicking on the logo at the top left, the user should be directed to this homepage. When clicking on “Kampoppsett” the user will be directed to fixtures.html. When clicking on “Lahet” the user will be directed to team.html. When hovering the different sections, they will become larger/bold.

fixtures.html

fixtures show information about all the matches of the team. The next match is presented at the top, like we did in the homepage. Every match is shown in a table, because it is an informative way to present data. If the match has been played, the results will be displayed. In this way it is easy to see the result of played matches, and when and where the upcoming matches will take place. As mentioned, the navigation bar is also present at this page.

team.html

This is the information page for the football team. The goal for this page is to inform the user about the team. The page provides a short descriptive text of the team, as well as a teamphoto and the logo. And the navigation bar is present at the top of the page.

Minimum requirements

Our objective is to meet the minimum requirements while also incorporating additional functionality. Below you can see how we intend to meet these requirements:

  • OnClick.js

    • This Javascript function is implemented to make something happen when the user clicks something. For example, if the user clicks the Nonstop logo, the user should be redirected to the home page. This function will also be used when the user wishes to show interest in a news article, or click on “send in”.

  • Hover.js

    • The “Hover.js” function should add interactive hover effects to enhance the user experience. These effects will be applied to the different menus: “About the team”, “Kampoppsett” and “Laget””. These effects should also be applied to buttons and links on the site, and will help make the website more engaging and user-friendly.

  • Popup.js

    • This function is used to create pop-up windows to display additional information when the user interacts with specific elements on the webpage. In our case, when a user wishes to show interest in a news article in the home page, there will pop-up showing the whole article.

  • NewsArticle.js

    • A function to help display the content of the news articles. Photo, Heading and text.
